# Hermes Desktop 使用文档

## 应用概述

Hermes Desktop 是 Hermes Agent 的桌面管理客户端，提供图形化界面来管理和监控 Hermes Agent 的运行状态。通过本应用，您可以：

- 启动/停止 Hermes Gateway 服务
- 查看和管理 AI 会话
- 监控系统状态和 Token 用量
- 配置模型参数和平台集成
- 管理定时任务和技能

## 连接模式

Hermes Desktop 支持两种连接模式：

### 本地模式（推荐）

本地模式下，Hermes Gateway 直接在您的本地计算机上运行。

- **适用场景**：个人使用、开发测试
- **优势**：无需网络连接，低延迟，数据完全本地化
- **要求**：需要在本地安装并配置 Hermes Agent

在本地模式下，应用会自动检测本地 Gateway 的运行状态，并在仪表盘上显示连接状态。

### 远程模式

远程模式下，Heremes Gateway 运行在远程服务器上，桌面客户端通过 SSH 隧道连接。

- **适用场景**：服务器部署、团队协作、24/7 运行
- **优势**：可以持续运行，不受本地计算机关机影响
- **要求**：需要 SSH 访问远程服务器

SSH 隧道建立后，本地端口会自动转发到远程服务器的 Gateway 端口，对用户来说就像访问本地服务一样。

## 设置步骤

### 首次使用设置

1. 打开应用，进入「设置」页面
2. 选择连接模式（本地或远程）
3. 如果选择远程模式，填写 SSH 配置信息
4. 可选：修改端口覆盖（默认 8642）
5. 配置 Gateway 地址和 API Key
6. 编辑 .env 文件配置环境变量
7. 点击「重启 Gateway」使配置生效

### 远程模式 SSH 配置

填写以下信息：

- **主机地址**：远程服务器的 IP 地址或域名
- **SSH 端口**：SSH 服务端口，默认 22
- **用户名**：SSH 登录用户名
- **密钥路径**：SSH 私钥文件路径（如 `~/.ssh/id_rsa`）

填写完成后，点击「测试连接」按钮验证 SSH 连接是否可用。

### 端口覆盖

默认情况下，Hermes Gateway 使用 8642 端口。如果需要使用其他端口，可以在端口覆盖字段中修改。修改后，应用会自动使用新端口进行连接。

## 仪表盘说明

仪表盘是应用的主页面，显示以下信息：

### 状态卡片

- **连接状态**：显示当前 Gateway 的连接状态（已连接/未连接/错误）
- **运行模式**：显示当前是本地模式还是远程模式
- **Gateway 地址**：当前使用的 Gateway URL

### 统计信息

- **总会话数**：历史 AI 会话总数
- **Token 用量**：累计消耗的 Token 数量
- **输入 Token**：输入模型的总 Token 数
- **输出 Token**：模型输出的总 Token 数
- **日均会话**：平均每天的会话数量

### 最近会话

按时间倒序显示最近的 AI 会话列表，每条会话显示：

- 会话时间
- 用户消息预览
- 最后回复预览
- 消息总数

点击会话可查看详细对话内容。

## 聊天功能

### 发起新会话

1. 在聊天页面输入消息
2. 按 Enter 发送（Shift+Enter 换行）
3. AI 会以流式方式实时回复

### 会话管理

- **查看历史**：在仪表盘或会话列表中选择历史会话
- **续传会话**：打开历史会话后，在最后继续发送消息即可续传
- **删除会话**：在会话详情中可以删除不需要的会话

### API Key 认证

如果配置了 API Key，所有请求会自动携带 Bearer Token 认证。API Key 需要与 `~/.hermes/.env` 中设置的 `API_SERVER_KEY` 值一致。

## 常见问题

### Q: 本地模式下显示"Gateway 未响应"

A: 请确保 Hermes Gateway 已正确安装并启动。可以尝试以下步骤：

1. 检查 `~/.hermes/config.yaml` 配置是否正确
2. 在终端运行 `hermes` 命令启动 Gateway
3. 在设置页面点击「重启 Gateway」
4. 检查端口 8642 是否被其他程序占用

### Q: 远程 SSH 连接失败

A: 请检查以下配置：

1. SSH 服务器地址和端口是否正确
2. 用户名和密钥路径是否正确
3. 服务器防火墙是否开放了 SSH 端口（默认 22）
4. 服务器上是否已安装并启动了 Hermes Gateway
5. 尝试在终端中手动使用 SSH 命令测试连接

### Q: 如何修改 Gateway 端口？

A: 在设置页面的「端口覆盖」字段中输入新的端口号，然后重启 Gateway 即可。如果使用远程模式，SSH 隧道会自动使用新的端口进行转发。

### Q: 如何在远程服务器上安装 Hermes Agent？

A: 请参考 Hermes Agent 官方文档，在远程服务器上执行安装脚本。确保安装完成后，Gateway 服务正常运行。

### Q: 会话续传不生效

A: 请确保：

1. 在 `~/.hermes/.env` 中正确设置了 `API_SERVER_KEY`
2. 在桌面应用的设置页面配置了相同的 API Key
3. Gateway 已重启使配置生效

### Q: 应用启动后显示空白

A: 可能是 Gateway 未启动或连接失败。请检查：

1. 终端中是否有 Hermes Gateway 进程运行
2. `~/.hermes/config.yaml` 是否存在且格式正确
3. 尝试清除桌面配置缓存：删除 `~/.hermes/desktop_config.json` 后重启应用

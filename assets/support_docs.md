# Hermes Desktop 使用文档

## 应用概述

Hermes Desktop 是 Hermes Agent 的桌面管理客户端，提供图形化界面来管理和监控 Hermes Agent 的运行状态。通过本应用，您可以：

- 管理 Hermes Gateway 连接（本地/内嵌/远程三种模式）
- 查看和管理 AI 会话（续传、删除、重命名）
- 与 AI 进行流式对话
- 浏览和管理服务器文件
- 查看系统日志并过滤搜索
- 管理定时任务（Cron Jobs）
- 配置模型参数和平台集成（Telegram、Discord、微信等）
- 浏览和管理技能（Skills）

## 连接模式

Hermes Desktop 支持三种连接模式，对外暴露的接口一致，内部实现根据模式不同采用不同的数据获取方式。

### 本地模式（WSL）

本地模式下，Hermes Gateway 运行在 WSL（Windows Subsystem for Linux）中，桌面客户端通过 `wsl.exe` 执行命令。

- **适用场景**：个人使用、开发测试
- **优势**：完整的 Linux 环境，无需额外配置
- **要求**：系统已安装 WSL 和 Hermes Agent
- **数据获取**：通过 WSL Bash 执行 Linux 命令

### 内嵌模式（Windows 原生）

内嵌模式下，Hermes 直接在 Windows 上原生运行，桌面客户端通过 `cmd.exe` 执行命令，文件操作使用 Windows 原生 API。

- **适用场景**：没有 WSL 的 Windows 环境
- **优势**：不依赖 WSL，直接运行
- **要求**：已安装 Hermes Windows 版本
- **数据获取**：通过 Windows 原生 API（dart:io）直接读写文件

### 远程模式（SSH）

远程模式下，Hermes Gateway 运行在远程 Linux 服务器上，桌面客户端通过 SSH 隧道连接。

- **适用场景**：服务器部署、团队协作、24/7 运行
- **优势**：可持续运行，不受本地计算机关机影响
- **要求**：SSH 访问远程服务器，远程服务器已安装 Hermes Agent
- **数据获取**：通过 SSH 执行远程命令

## 首次使用设置

### 首次启动流程

1. 应用首次启动时会自动检测 WSL 环境
2. 如果检测到 WSL 中存在 Hermes，自动选择本地模式
3. 如果未检测到，弹出设置向导让您选择模式
4. 选择连接模式后，应用自动尝试连接
5. 连接成功后进入主界面

### 手动切换模式

在「设置」页面的「连接模式」区域选择所需模式：

- **本地模式**：自动使用 WSL 中的 Hermes
- **内嵌模式**：使用 Windows 原生的 Hermes
- **远程模式**：需要配置 SSH 连接信息

切换模式后，所有页面会自动刷新以显示新模式下的数据。

## 远程模式 SSH 配置

填写以下信息：

- **主机地址**：远程服务器的 IP 地址或域名
- **SSH 端口**：SSH 服务端口，默认 22
- **用户名**：SSH 登录用户名
- **密码**：SSH 登录密码（密码认证）
- **密钥路径**：SSH 私钥文件路径，如 `~/.ssh/id_rsa`（密钥认证）

填写完成后，点击「测试连接」按钮验证 SSH 连接是否可用。

SSH 隧道建立后，本地端口自动转发到远程服务器的 Gateway 端口，对用户来说就像访问本地服务一样。

## 仪表盘

仪表盘是应用的主页，显示以下信息：

### 状态卡片

- **当前模型**：正在使用的 AI 模型名称
- **技能数量**：已安装的技能总数
- **会话数量**：历史 AI 会话总数
- **日志大小**：Agent 日志占用的磁盘空间

### 连接信息

顶部状态栏显示当前连接模式（本地/内嵌/远程）、连接状态（在线/离线）和 Gateway 地址。

### 最近会话

按时间倒序显示最近的 AI 会话列表，每条会话显示：

- 会话时间
- 用户消息预览
- 最后回复预览
- 消息总数

点击会话可查看详细对话内容。

## 聊天功能

### 发起新会话

1. 在聊天页面输入消息
2. 按 Enter 发送（Shift+Enter 换行）
3. AI 以流式方式实时回复

### 会话管理

- **查看历史**：左侧会话列表选择历史会话
- **续传会话**：打开历史会话后，继续发送消息即可续传
- **删除会话**：在会话菜单中可删除不需要的会话
- **重命名会话**：在会话菜单中可重命名会话

### 图片附件

发送消息时可附带图片，支持 PNG、JPG 等常见格式。

### API Key 认证

如果配置了 API Key，所有请求会自动携带 Bearer Token 认证。API Key 需要与 `~/.hermes/.env` 中设置的 `API_SERVER_KEY` 值一致。

## 文件浏览器

文件浏览器以用户 Home 目录为根目录，支持：

- 浏览目录结构（文件和文件夹分别排序）
- 预览文本文件内容（右侧预览面板）
- **本地模式**：在 Windows 资源管理器中打开文件位置
- **远程模式**：下载文件到本地

## 日志查看器

日志页面提供三种日志标签页：

- **Agent 日志**：Hermes Agent 的运行日志
- **Gateway 日志**：Gateway 服务的访问日志
- **错误日志**：仅显示错误级别的日志

### 日志功能

- **级别过滤**：ALL / INFO / WARN / ERROR / DEBUG
- **关键词搜索**：按关键字过滤日志内容
- **自动滚动**：新日志自动滚动到底部
- **清空显示**：清空当前显示（不影响文件）
- **删除日志**：清除日志文件内容
- **复制日志**：右键点击日志条目可复制内容

## 定时任务（Cron）

定时任务页面用于管理 Hermes 的定时任务：

- **任务列表**：显示所有已配置的任务
- **创建任务**：选择技能、设置调度表达式、填写提示词
- **编辑任务**：修改现有任务的参数
- **暂停/恢复**：临时启用或禁用任务
- **删除任务**：移除不需要的任务

任务通过 `hermes cron` 命令执行，调度表达式使用标准的 Cron 格式（如 `0 9 * * *` 表示每天 9:00）。

## 平台管理

平台管理页面用于配置 Hermes 与外部平台的集成：

- **支持平台**：Telegram、Discord、Slack、WhatsApp、飞书、企业微信、Matrix、微信个人号、Signal、Email、钉钉、QQ Bot
- **配置状态**：显示各平台是否已配置
- **连接状态**：显示各平台的运行状态（已连接/已断开）
- **微信配置**：支持扫码登录流程（本地模式）

### 配置平台

1. 点击平台卡片进入详情
2. 根据提示填写必要的配置信息
3. 配置信息保存在 `~/.hermes/.env` 中
4. 保存后自动重启 Gateway 使配置生效

## 技能管理

技能是 Hermes 的功能扩展包，每个技能包含一个 `SKILL.md` 描述文件。

- **技能列表**：显示所有已安装的技能
- **技能详情**：查看技能的名称、描述、作者等信息
- **安装位置**：`~/.hermes/skills/` 目录下

## 模型配置

模型页面用于查看和配置 AI 模型：

- **当前模型**：显示正在使用的模型
- **模型列表**：查看所有可用的模型
- **提供商配置**：配置不同模型提供商的信息

## 常见问题

### Q: 本地模式下显示"Gateway 未响应"

A: 请确保 WSL 中 Hermes Gateway 已正确安装并启动：

1. 检查 `~/.hermes/config.yaml` 配置是否正确
2. 在 WSL 终端运行 `hermes` 命令启动 Gateway
3. 在设置页面点击「重启 Gateway」
4. 检查端口 8642 是否被其他程序占用

### Q: 内嵌模式下无法获取数据

A: 内嵌模式使用 Windows 原生文件 API，请确保：

1. Hermes Windows 版本已正确安装
2. `%USERPROFILE%\.hermes\` 目录存在且包含正确的配置文件
3. 如果 Hermes 尚未安装，请先从 GitHub Releases 下载安装

### Q: 远程 SSH 连接失败

A: 请检查以下配置：

1. SSH 服务器地址和端口是否正确
2. 用户名和密码/密钥路径是否正确
3. 服务器防火墙是否开放了 SSH 端口（默认 22）
4. 服务器上是否已安装并启动了 Hermes Gateway
5. 尝试在终端中手动使用 SSH 命令测试连接

### Q: 如何修改 Gateway 端口？

A: 在设置页面的「Gateway 端口」字段中输入新的端口号，保存后重启 Gateway 即可。远程模式下 SSH 隧道会自动使用新端口转发。

### Q: 如何在远程服务器上安装 Hermes Agent？

A: 请参考 Hermes Agent 官方文档，在远程服务器上执行安装脚本。确保安装完成后 Gateway 服务正常运行。

### Q: 切换连接模式后数据不刷新？

A: 切换模式后所有页面会自动监听连接状态变化并刷新数据。如果某个页面未自动刷新，可手动点击页面上的刷新按钮，或重新进入该页面。

### Q: 应用启动后显示空白

A: 可能是 Gateway 未启动或连接失败：

1. 检查 Gateway 是否正在运行
2. `~/.hermes/config.yaml` 是否存在且格式正确
3. 尝试清除桌面配置缓存：删除 `~/.hermes/desktop_config.json` 后重启应用

### Q: 文件浏览器显示为空或错误

A: 文件浏览器默认打开用户 Home 目录。如果显示为空：

1. 检查当前连接模式是否正确
2. 点击右上角的刷新按钮
3. 点击 Home 按钮重新加载根目录
4. 查看日志页面是否有相关错误信息
